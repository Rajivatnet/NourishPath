# Phase 2: Technical foundation

The MVP starts with a single landing and demo sign-in page. The API includes a health endpoint and a demo authentication endpoint for `Guest1`–`Guest3`. The hardcoded `ADMIN` password is strictly a hackathon shortcut and must be replaced by a proper identity provider or securely stored password hashes before production.

Future phases can add patient goals, meal recommendations, nutrition education, and carefully vetted external shopping links. The product must clearly avoid diagnosing or promising treatment outcomes.
